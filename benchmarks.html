Methodology & caveats

  • Read-side benchmark — measures context retrieval, not code generation quality
  • Databricks SDK workload — real-world codebase exploration tasks
  • No broad quality scoring in this run — focus was on context efficiency
  • Write-side tools excluded — edit/test/verify loop benchmarked separately
  • Quality benchmark for edit/test/verify tasks is a separate evaluation

Comparable answer with dramatically less context and tool usage. Not "always better answers."
