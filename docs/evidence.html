What are evidence packs?

Evidence packs are structured, machine-readable outputs from Lakecode workflows. They capture the data quality metrics and diagnostics that the AI gathered during execution, formatted as standalone artifacts you can share, attach, or archive.

The /prove evidence pack

When you run /prove on a table, Lakecode generates an evidence pack containing:

  • Schema snapshot — Column names, types, and nullability as of the check time
  • Row count — Total rows in the table
  • Null analysis — Per-column null counts and percentages
  • Freshness check — When the table was last updated, how stale it is
  • Duplicate detection — Checks for duplicate rows on key columns
  • Numeric statistics — Min, max, mean, stddev for numeric columns
  • Sample rows — A small sample of actual data for visual inspection
  • Health assessment — An overall pass/warn/fail verdict with explanations

Output location

Evidence packs are saved to the ./evidence/ directory in your current working directory. Each pack is timestamped and named after the table.

Use cases

  • Pull requests — Attach evidence packs to PRs that modify table schemas or ETL logic as proof that downstream data is healthy.
  • Incident response — Attach to incident tickets to document the state of affected tables at the time of investigation.
  • Audit trails — Keep a history of data quality checks for compliance and governance.
  • Stakeholder communication — Share structured metrics with non-technical stakeholders instead of raw query output.
