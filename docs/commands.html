All commands are invoked as slash commands within the Lakecode CLI. Each command connects to your Databricks or Snowflake workspace using your existing CLI profile and auth.

Commands labeled Databricks or Snowflake are platform-specific. Unlabeled commands work on both platforms.

Observability

/status job <id> Databricks

Retrieves the job configuration and its last 5 runs. Use this as a quick health check for any job — see the schedule, cluster config, and recent run outcomes at a glance.

lakecode /status job 123456
/logs <run-id> Databricks

Fetches the full output of a specific run, including stdout, stderr, and error tracebacks. Lakecode analyzes the errors and highlights the root cause.

lakecode /logs 78901234
/debug job <id> Databricks

Full diagnosis workflow: pulls the job config, finds the most recent failed run, extracts errors, checks query history, and produces a root cause analysis. This is the single command for “why is this job broken?”

lakecode /debug job 123456
/debug task <name> Snowflake

Diagnose a failed Snowflake task: fetches the task definition, run history, error details, and produces a root cause analysis.

lakecode /debug task DAILY_REFRESH
/queries job <id> Databricks

Retrieves the SQL query history for the latest run of a job. Useful for understanding what queries a job executed and how long each took.

lakecode /queries job 123456
/queries run <run-id> Databricks

Retrieves the SQL query history for a specific run. Use this when you need to inspect queries from a particular execution, not just the latest.

lakecode /queries run 78901234
/watch job <id> Databricks

Subscribe to real-time monitoring of a job. Receives alerts on failures, cost anomalies, and SLA breaches. Use /watch list to see active subscriptions, /watch stop to cancel.

lakecode /watch job 123456

Data Quality

/prove <table>

Generates a comprehensive evidence pack for a table: schema snapshot, row count, null analysis, freshness check, duplicate detection, numeric statistics, sample rows, and an overall health assessment. Results are saved to the ./evidence/ directory. Works on both Databricks and Snowflake.

lakecode /prove my_catalog.my_schema.my_table
/audit jobs Databricks

Scans all jobs in your workspace for SLA gaps, missing timeout configurations, and recurring failures. Produces a risk-scored report so you can prioritize which jobs need attention.

lakecode /audit jobs
/audit tasks Snowflake

Audit all Snowflake tasks: checks ownership, schedules, failure rates, and generates a compliance report.

lakecode /audit tasks

Cost Analysis

/cost top

Queries your platform billing tables and identifies the top spend drivers. On Databricks: jobs, clusters, and warehouses. On Snowflake: warehouses and queries.

lakecode /cost top
/cost spike

Compares a target day’s spend against a baseline period to detect cost spikes. Identifies the specific drivers behind any increase. Works on both platforms.

lakecode /cost spike
/genai ask <question> Databricks

GenAI cost monitoring sub-agent. Answers questions about GenAI spend using Unity Catalog functions — e.g., “which endpoints cost the most last week?”

lakecode /genai ask "what is our top GenAI spend driver?"

Governance & Unity Catalog

/uc explain-access <principal> <object> Databricks

Trace why a principal has (or lacks) access to a Unity Catalog object by walking the grant hierarchy. Supports an optional --action flag (e.g., SELECT).

lakecode /uc explain-access user@company.com analytics.prod.sales
/uc diff-grants <object> --to <desired.yml> Databricks

Compare current UC grants against a desired-state YAML and produce a reviewable plan showing what needs to be added, removed, or changed.

lakecode /uc diff-grants analytics.prod --to grants.yml
/uc apply-grants --plan <planId> Databricks

Execute a previously generated grant plan with before/after snapshots for audit.

lakecode /uc apply-grants --plan abc123

Deployment & Bundles

/deploy <local-path> <workspace-path> Databricks

Imports a local file (notebook, SQL, Python) into your Databricks workspace at the specified path.

lakecode /deploy ./notebooks/etl.py /Workspace/production/etl
/run job <id> Databricks

Triggers a job run and polls for completion. Shows real-time status updates until the run finishes or fails.

lakecode /run job 123456
/capture job <id> Databricks

Capture a Databricks job definition into a DAB (Databricks Asset Bundle) configuration. Also supports /capture pipeline <id> for DLT pipelines.

lakecode /capture job 123456 --to ./bundle
/drift detect Databricks

Compare deployed Databricks resources against your bundle definitions to detect configuration drift. Flags any differences between what’s defined in code and what’s live.

lakecode /drift detect --target prod

Session & Utility

/model <name>

Switch the AI model mid-session. Supports opus, sonnet, and haiku for different speed/quality tradeoffs.

lakecode /model sonnet
/sessions list

List saved conversation sessions. Use /sessions delete <id> to remove old sessions.

/context

Show the current workspace context: platform, profile, catalogs, schemas, and active conventions.

/replay

Replay a previous session or workflow run for review or re-execution.

/help

Show the full list of available commands and usage information.
