Governed workflows with approval gates

Every AI action is classified, logged, and gated — so your team moves fast without losing control.

Safety classification

  • Every operation classified as READ, WRITE, or DESTRUCTIVE
  • WRITE and DESTRUCTIVE actions require explicit approval
  • Configurable per org — set your own risk thresholds

Approval gates

  • Human-in-the-loop before any state-changing operation
  • Step manifests show exactly what will execute and why
  • Approve, reject, or modify before the agent proceeds

Evidence packs & audit trails

  • Every workflow produces structured evidence artifacts
  • Full execution logs tied to user identity and timestamp
  • Attach to PRs, incidents, compliance reviews, or Jira tickets

AssistantSpec — configure per org

  • Define which workflows are available to your team
  • Set default catalogs, schemas, naming conventions
  • Enforce org-specific policies and guardrails

Databricks-native deployment

Databricks Apps

Deploy lakecode as a Databricks App with a managed UI. Your team gets governed AI workflows without installing anything locally.

Model Serving

Run the agent backend on Databricks Model Serving for low-latency, scalable execution inside your cloud perimeter.

Unity Catalog integration

UC-backed state store for execution history, evidence packs, and configuration. Leverage existing access controls and lineage.

Platform architecture

Built for security, auditability, and your existing Databricks investment.

Serverless workspace provisioning

Ephemeral, isolated compute for each workflow execution. No long-running clusters, no shared state between runs.

Step manifests

Every workflow decomposes into typed steps with declared inputs, outputs, and safety classification. Full transparency before execution.

Model governance via AI Gateway

Route all LLM calls through Databricks AI Gateway. Enforce rate limits, logging, and model selection at the org level.

UC-backed state store

Execution history, evidence artifacts, and agent configuration stored in Unity Catalog. Governed by your existing access policies.
